# 코드 리뷰 프로토콜 (코드 작성 후)

`src/` 의 파일을 작성하거나 의미 있게 수정한 후,
**항상** 완료 보고 전에 코드를 리뷰하세요.

## 트리거 시점
- `src/`에 새 파일 생성
- 기존 파일의 의미 있는 수정 (모델, 데이터셋, 트레이너, 스크립트, 설정)

## 확인 항목

**코드 구조**
- Import 경로가 실제 모듈 위치와 일치
- 절대/상대 경로 혼합 오류 없음
- Config YAML 키가 코드에서 참조하는 키와 일치
- 누락된 `__init__.py` 또는 미등록 모듈 없음

**모델 설정**
- Gradient flow가 깨지지 않음
- 가중치 초기화가 존재하고 올바름
- Loss reduction 모드 (`sum` vs `mean`)가 적절
- BatchNorm/LayerNorm train/eval 모드가 올바르게 처리됨
- Forward pass 전체에서 텐서 shape가 일관적
- Mixed-precision 설정에서 dtype 충돌 없음

## 보고
- 이슈 발견 시 → 수정한 후, 수정 내용 요약
- 이슈 없음 → 보고: `코드 리뷰 통과: 구조적 또는 모델 설정 이슈 없음.`
