# 안전 제약 규칙 (AI 연구 공통)

## 절대 규칙

1. **프로젝트 경로 제한**: 모든 파일 생성/수정/삭제는 반드시 프로젝트 루트 내부에서만 수행
2. **rm -rf 금지**: 절대 사용하지 않음. 디렉토리 삭제가 필요하면 연구자에게 컨펌 요청
3. **루트 경로 보호**: `/` 바로 하위 경로(예: `/root`, `/home`, `/etc`, `/usr`)를 직접 수정하지 않음
4. **데이터 백업 보호**: `data_backup/`은 어떤 상황에서도 절대 수정하지 않음
5. **삭제 컨펌 필수**: 파일/디렉토리 삭제 시 반드시 연구자에게 삭제 대상 목록을 보여주고 승인 후 실행

## 학습 안전 규칙

6. **checkpoint 보호**: 기존 학습된 best_model.pth / checkpoint_best.pth를 덮어쓰지 않음
7. **dry-run 격리**: dry-run 모드에서 training_complete.flag를 생성하지 않음
8. **학습 skip 로직 필수**: 학습 스크립트에는 기존 checkpoint 존재 시 skip 로직을 포함

## 실행 환경 규칙

9. **conda 환경**: 모든 Python 실행은 지정된 conda 환경에서 수행
10. **GPU 검증**: 학습 시작 전 GPU 가용성 및 개수 확인 (DDP 시 2개 이상)
11. **GPU 점유 확인**: 공유 서버에서 다른 사용자의 GPU 점유 확인 후 컨펌 필수
12. **tmux 사용**: 장시간 학습은 반드시 tmux 세션 내에서 실행 (연결 끊김 대비)
