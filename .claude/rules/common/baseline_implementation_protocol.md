# Baseline 모델 구현 프로토콜

## 원칙
Baseline 모델은 원논문의 구조를 **최대한 충실하게 재현**하는 것이 목표입니다.
공정 비교를 위해 임의 변형 없이 원논문 그대로 구현합니다.

## 구현 전 필수 절차

### 1단계: 원논문 정독
- 해당 baseline 논문을 **전체** 꼼꼼히 읽을 것
- Method, Supplementary, Appendix, 공개 코드까지 확인

### 2단계: 분석 보고서 작성 후 컨펌
구현 전에 반드시 다음 내용을 정리하여 **연구자에게 보여주고 승인을 받을 것**:

1. **모델 아키텍처 요약** — 파이프라인, stage별 구조, 입출력 형태
2. **학습 설정** — Loss, Optimizer, LR schedule, Augmentation, Batch size
3. **논문과 우리 환경의 차이점** — 데이터 차이, 조정이 필요한 부분
4. **구현 계획** — 파일 구조, 핵심 클래스/함수 목록

### 3단계: 컨펌 후 구현
연구자가 분석 보고서를 확인하고 승인한 후에만 코드 작성을 시작합니다.

## 주의사항
- 원논문에 명시되지 않은 구현 세부사항은 **추측하지 말고 질문**할 것
- 성능 향상을 위한 임의 수정 금지
- 원논문 공개 코드가 있으면 반드시 참조하되, 라이선스 확인 필수
